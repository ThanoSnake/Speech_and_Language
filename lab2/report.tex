\subsection{EX2 --- \texttt{SentenceDataset.\_\_init\_\_}}

The dataset constructor tokenizes each raw sentence using NLTK's
\texttt{TweetTokenizer} (chosen for its handling of Twitter-specific tokens
such as hashtags, mentions, and emoticons) and maps each token to its index
in the pre-trained embedding vocabulary.
Out-of-vocabulary words are mapped to the \texttt{<unk>} token, which is
added to \texttt{word2idx} with a random vector by \texttt{load\_word\_vectors}
if not already present in the embeddings file.

\begin{lstlisting}
tokenizer = TweetTokenizer()
self.data = [tokenizer.tokenize(sentence) for sentence in X]
self.embedded_data = [
    [word2idx.get(word, word2idx["<unk>"]) for word in sentence]
    for sentence in self.data
]
self.labels = y
self.word2idx = word2idx
\end{lstlisting}

\subsection{EX3 --- \texttt{SentenceDataset.\_\_getitem\_\_}}

The \texttt{\_\_getitem\_\_} method returns a tuple of three values for a given index:

\begin{enumerate}
    \item \textbf{example} --- the padded/truncated sequence of word indices
    \item \textbf{label} --- the integer class label
    \item \textbf{length} --- the real number of tokens, excluding zero-padding
\end{enumerate}

\begin{lstlisting}
def __getitem__(self, index):
    embeddings = self.embedded_data[index]
    length = min(len(self.data[index]), self.max_length)
    label = self.labels[index]
    return embeddings, label, length
\end{lstlisting}

The \texttt{max\_length} was set to \textbf{40} after inspecting the sentence length
distribution in the training set. Sentences shorter than 40 tokens are zero-padded;
longer sentences are truncated.

The \texttt{length} is capped at \texttt{max\_length} because sentences that exceed
it are truncated in \texttt{embedded\_data} --- returning the pre-truncation token
count would be inconsistent with the actual stored sequence.

Each example passes through three stages of processing, shown below for 3
representative samples from the MR training set (max\_length = 40):

\section{Model}

\subsection{EX4 --- Embedding Layer}

\begin{lstlisting}
# 1 - define the embedding layer
self.embedding = nn.Embedding(len(embeddings), len(embeddings[0]))

# 2 - initialize weights from pretrained word embeddings
self.embedding.weight = nn.Parameter(torch.Tensor(embeddings))

# 3 - freeze or finetune depending on trainable_emb
self.embedding.weight.requires_grad = trainable_emb
\end{lstlisting}

\textbf{Why initialize from pre-trained embeddings?}
Pre-trained word embeddings such as GloVe are trained on massive corpora (billions of tokens)
and encode rich semantic and syntactic relationships between words. Initializing
the embedding layer with these vectors gives the model a high-quality starting point
that would be impossible to learn from our comparatively small sentiment datasets alone.

\textbf{Why freeze the embedding layer during training?}
The pre-trained embeddings already form a well-structured, expressive vector space in which
semantically related words lie close together. The embedding layer contains far more
parameters than the rest of the classifier, and our sentiment datasets are comparatively
small. Allowing gradients to update these parameters risks distorting the global structure
of the embedding space to fit the idiosyncrasies of the training set---in other words,
overfitting. The small task-specific dataset simply does not carry enough information to
justify re-tuning hundreds of thousands of embedding parameters, and doing so is likely to
degrade the quality of the representations rather than improve them.

\subsection{EX5 --- Output Layers}

\begin{lstlisting}
hidden_size = 1024
self.fc1 = nn.Linear(emb_dim, hidden_size)
self.ac1 = nn.ReLU()
self.output_layer = nn.Linear(hidden_size, output_size)
\end{lstlisting}

\textbf{Why do we place a non-linear activation before the output layer?
What would change if we stacked two or more linear transformations in sequence?}

A composition of linear transformations is itself linear:
$\mathbf{y} = W_2\,(W_1\,\mathbf{x}) = (W_2 W_1)\,\mathbf{x}$.
The product $W_2 W_1$ is a single matrix, so no matter how many linear layers we chain
together, the set of representable functions $\mathcal{H}$ remains the set of all affine
maps---the same as a single linear layer with fewer parameters. The optimal achievable
loss $L^{*} = \min_{f \in \mathcal{H}} \|f - f^{*}\|$ therefore stays the same; we only
add redundant parameters without expanding the model's expressive power.

Inserting a non-linear activation such as ReLU between the layers breaks this equivalence.
The resulting network can represent non-linear decision boundaries, strictly enlarging
$\mathcal{H}$ and potentially lowering $L^{*}$.

\subsection{EX6 --- Forward Pass}

\begin{lstlisting}
# 1 - embed the words
embeddings = self.embedding(x)

# 2 - mean of word embeddings (excluding padding)
representations = torch.sum(embeddings, dim=1) \
                  / lengths.unsqueeze(1).float()

# 3 - non-linear transformation
representations = self.ac1(self.fc1(representations))

# 4 - project to classes
logits = self.output_layer(representations)
\end{lstlisting}

\textbf{Intuitive interpretation of the centroid representation.}
If we assume that each dimension of the embedding space captures an abstract semantic
property, then the centroid (mean) of all word embeddings in a sentence represents the
``average semantic profile'' of that sentence along every such dimension.
For example, suppose one dimension correlates with \emph{sentiment intensity}. In the
sentence \emph{``a thrilling, breathtaking masterpiece''}, the words \emph{thrilling},
\emph{breathtaking}, and \emph{masterpiece} all project strongly onto this axis, so the
centroid inherits a high value on that dimension---capturing that the sentence as a whole
expresses strong positive sentiment.

\textbf{Weaknesses of the centroid approach.}

\begin{itemize}
    \item \textbf{Word order is lost.} The centroid is invariant to permutations of the
    input words: \emph{``the movie was not expensive''} and \emph{``expensive was not the
    movie''} yield identical representations. More critically, negation words such as
    \emph{not} are often nearly orthogonal to the property they negate. Consider
    \emph{``the movie was expensive''} vs.\ \emph{``the movie was not expensive''}:
    the word \emph{not} likely has a near-zero projection onto a ``cost'' dimension,
    so the centroids of the two sentences land very close together despite carrying
    opposite meanings.

    \item \textbf{Context is ignored.} Each word receives the same embedding regardless of
    its surrounding context. The word \emph{bank} gets the same vector whether it appears
    in \emph{``river bank''} or \emph{``bank account''}---the centroid has no mechanism to
    disambiguate polysemous words.
\end{itemize}

\section{Training}

\subsection{EX7 --- DataLoaders}

\begin{lstlisting}
train_loader = DataLoader(train_set, batch_size=BATCH_SIZE, shuffle=True)
test_loader  = DataLoader(test_set,  batch_size=BATCH_SIZE)
\end{lstlisting}

\textbf{What are the consequences of small vs.\ large mini-batches?}

The per-batch loss $L_B = \frac{1}{|B|}\sum_{i \in B} \ell_i(\theta)$ is a sample mean
over i.i.d.\ draws, so its variance is:
\[
    \mathrm{Var}(L_B) = \frac{\sigma_\ell^2}{|B|}
\]
where $\sigma_\ell^2$ is the per-sample loss variance. This variance is an indicator of the ``jaggedness'' of the loss curve: higher variance
means larger fluctuations in the reported loss from batch to batch. Since it decreases
with $|B|$, this explains why:

\begin{itemize}
    \item \textbf{Small batches} have high $\mathrm{Var}(L_B)$, so the loss curve
    appears jagged. However, they yield more weight updates per epoch and the gradient
    noise acts as a regularizer---it prevents the model from settling into sharp minima,
    reducing overfitting.

    \item \textbf{Large batches} have low $\mathrm{Var}(L_B)$, leading to a smooth loss
    curve and more predictable convergence. The downside is fewer updates per epoch and a
    tendency to converge to sharper minima that may generalize worse.
\end{itemize}

\textbf{Why do we shuffle the training data each epoch?}

Without shuffling, the same examples always form the same batches and appear in the same
order. This introduces \emph{sequential bias}: the weight updates at the end of each
epoch are systematically driven by the same subset of examples, skewing the model toward
patterns in that subset. For instance, if the last batches happen to contain mostly
negative reviews, the final updates of every epoch would push the model toward predicting
``negative.'' Shuffling breaks this fixed ordering, ensures different batch compositions
each epoch, and makes the gradient estimates more representative of the full dataset
across training.

\subsection{EX8 --- Loss Function, Parameters, and Optimizer}

\begin{lstlisting}
if DATASET == "MR":
    criterion = nn.BCEWithLogitsLoss()
    output_dim = 1  # single logit for binary classification
elif DATASET == "Semeval2017A":
    criterion = nn.CrossEntropyLoss()
    output_dim = 3  # 3 classes: positive, negative, neutral

model = BaselineDNN(
    output_size=output_dim,
    embeddings=embeddings,
    trainable_emb=EMB_TRAINABLE,
)

parameters = [p for p in model.parameters() if p.requires_grad]
optimizer = torch.optim.Adam(parameters, lr=0.001)
\end{lstlisting}

\textbf{Loss function.}
For the binary MR task (2 classes) we use \texttt{BCEWithLogitsLoss}, which combines a
sigmoid activation with binary cross-entropy in a single numerically stable operation.
The output layer has a single neuron (\texttt{output\_dim=1}), producing one logit;
the predicted class is positive if the logit exceeds zero.
For the three-class Semeval2017A task we use \texttt{CrossEntropyLoss}, which applies
softmax internally and computes the negative log-likelihood of the correct class.

\textbf{Trainable parameters.}
We collect only parameters with \texttt{requires\_grad == True}. When the embedding layer
is frozen (\texttt{trainable\_emb=False}), its weights are excluded from the optimizer,
leaving only the hidden layer and output layer weights to be updated. This avoids wasting
computation on frozen parameters and, more importantly, ensures the pre-trained embedding
space is not distorted by gradient updates from our small dataset.

\textbf{Optimizer.}
We use Adam, which maintains per-parameter adaptive learning rates via running estimates
of the first and second moments of the gradient. This makes it well-suited for our setting
where different parameter groups (hidden layer vs.\ output layer) may have very different
gradient magnitudes, removing the need to hand-tune a learning rate schedule.

\section{Ερώτημα 1 --- Mean + Max Pooling}

\subsection{1.1 --- Υλοποίηση}

Η αναπαράσταση κάθε πρότασης υπολογίζεται ως η συνένωση (concatenation) του
mean pooling και του max pooling των word embeddings:
\[
    \mathbf{u} = [\operatorname{mean}(E) \,\|\, \operatorname{max}(E)]
\]

Στον κώδικα, αυτό επιτυγχάνεται μέσω της παραμέτρου \texttt{max\_concat}
στο \texttt{BaselineDNN}:

\begin{lstlisting}
if self.max_concat:
    max_emb = torch.max(embeddings, dim=1).values
    representations = torch.cat(
        (representations, max_emb), dim=1
    )
\end{lstlisting}

Ο πρώτος γραμμικός μετασχηματισμός αλλάζει μέγεθος εισόδου ανάλογα:
\begin{lstlisting}
self.fc1 = nn.Linear(
    (2 if max_concat else 1) * emb_dim, hidden_size
)
\end{lstlisting}

\subsection{1.2 --- Σύγκριση με την αρχική αναπαράσταση}

Η αρχική αναπαράσταση (mean pooling μόνο) κρατάει μόνο τη ``μέση σημασιολογική
εικόνα'' της πρότασης: κάθε διάσταση του τελικού διανύσματος αντιστοιχεί στον
μέσο όρο αυτής της διάστασης πάνω σε όλα τα tokens.

Η προσθήκη του max pooling εξάγει παραπάνω πληροφορία:

\begin{itemize}
    \item \textbf{Ανίχνευση κυρίαρχων χαρακτηριστικών.}
    Το max pooling επιλέγει, ανά διάσταση, τη μέγιστη τιμή μεταξύ
    όλων των λέξεων. Αυτό σημαίνει ότι αρκεί \emph{μία} λέξη με
    ισχυρή ενεργοποίηση σε μια διάσταση (π.χ.\ μια έντονα αρνητική
    λέξη σε μια ``sentiment intensity'' διάσταση) για να αποτυπωθεί
    στην αναπαράσταση, ακόμα κι αν οι υπόλοιπες λέξεις δεν συνεισφέρουν
    σε αυτήν---κάτι που ο μέσος όρος θα ``αραίωνε''.

    \item \textbf{Συμπληρωματικότητα.}
    Ο μέσος όρος αποτυπώνει το ``γενικό τόνο'' της πρότασης, ενώ
    το μέγιστο αποτυπώνει τα ``ακραία σήματα''. Η συνένωσή τους
    δίνει στο μοντέλο πρόσβαση και στις δύο πληροφορίες, αυξάνοντας
    την εκφραστικότητα της αναπαράστασης χωρίς να εισάγει επιπλέον
    εκπαιδεύσιμες παραμέτρους στο embedding (μόνο ο πρώτος γραμμικός
    μετασχηματισμός μεγαλώνει).
\end{itemize}
