🔧 2. Improve Data Loading (Use Dataset + DataLoader)
❌ Original

from torch.utils.data import Dataset, DataLoader

class TextDataset(Dataset):
    def __init__(self, data, block_size):
        self.data = data
        self.block_size = block_size

    def __len__(self):
        return len(self.data) - self.block_size

    def __getitem__(self, idx):
        x = self.data[idx:idx+self.block_size]
        y = self.data[idx+1:idx+self.block_size+1]
        return x, y

train_dataset = TextDataset(train_data, block_size)
train_loader = DataLoader(train_dataset, batch_size=batch_size, shuffle=True)
💡 Why this is better
Enables:
multiprocessing (num_workers)
streaming large datasets
Cleaner abstraction
Standard PyTorch pipeline
🔧 3. Add Mixed Precision Training (Huge Speedup)

🔧 4. Safer & Cleaner Generation (Context Cropping)
❌ Original

(This part is actually good, but we can improve sampling)

✅ Refactored (with temperature + top-k)
def generate(self, idx, max_new_tokens, temperature=1.0, top_k=None):
    for _ in range(max_new_tokens):
        idx_cond = idx[:, -block_size:]
        logits, _ = self(idx_cond)

        logits = logits[:, -1, :] / temperature

        if top_k is not None:
            v, _ = torch.topk(logits, top_k)
            logits[logits < v[:, [-1]]] = float('-inf')

        probs = F.softmax(logits, dim=-1)
        idx_next = torch.multinomial(probs, num_samples=1)

        idx = torch.cat((idx, idx_next), dim=1)

    return idx
💡 Why this is better
More stable generation
Control randomness:
temperature
top-k sampling

👉 This is how real LLM APIs work
